todo.rb

Intro

todo.rb is a command-line todo list manager. It stores data as simple lists in flat text files and is highly interoperable with other Unix-style tools.

todo.rb is inspired by todo.sh by Gina Trapani, and takes it further in the direction of the Unix philosophy.

Like Merlin Mann, I keep todo lists in plain text files and used Vim to review and edit them. But Vim, like every popular text editor, presents a captive user interface which removes you from the command line and the immediate context of whatever you were doing there. Vim is also more difficult to script than a standard, noninteractive Unix tool.

todo.rb is a todo list tool with an unobtrusive, noncaptive, Unix-style interface. It leverages Unix filters to let you view your tasks by context or project. It uses ed under the hood to let you perform a wide range of edits on your todo.txt without leaving the command line.

Requires

Ruby 1.9

Install

$ gem install todo.rb

Basics

The full command name is todo.rb, but you'll probably want to alias it to something shorter. Put something like this in your ~/.bash_profile or ~/.bashrc: 

alias t=todo.rb

This is a detailed guide to the program. You can type this to get help from the command line: 

t -h

todo.rb operates in the current working directory. It looks for a todo.txt and done.txt file, and creates them if they don't already exist. 

$ t
Missing a todo.txt file. Creating.
Missing a done.txt file. Creating.

You can start adding tasks like this. 

$ t @harvardsq buy some assam from tealuxe
0a1
> @harvardsq buy some assam from tealuxe

@harvardsq is a context tag for your task. A context tag usually indicates a location, but it can also represent an activity, like @email. The two lines you see after the command prompt is a diff showing the change you just made.

To see the tasks we have accumulated so far, just press t. 

$ t
     1  @harvardsq buy some assam from tealuxe

If you look in the file todo.txt, you'll see that all todo.rb is doing besides printing the file is adding line numbers and coloring the context tags.

There is another type of tag you can use when you add tasks: a project tag. It looks like this: 

$ t +opensource release todo.rb
1a2
> +opensource release todo.rb

A project tag need not represent an actual project. It can stand for a topic, a feature, or whatever.

When you print the task list, you'll see that project tags and context tags are colored differently. You can also have more than one tag per task, and you can mix project and context tags. 

$ t
     1	@harvardsq buy some assam from tealuxe
     2	+opensource release todo.rb

Let's add one more task to the @harvardsq context. 

$ t @harvardsq buy more moleskine notebooks
2a3
> @harvardsq buy more moleskine notebooks

$ t
     1	@harvardsq buy some assam from tealuxe
     2	+opensource release todo.rb
     3	@harvardsq buy more moleskine notebooks

So when you type t [tag] [task] a new task will be created and tagged. If you don't want to provide a tag, or if you want to put the tag after the first word in your task, or if you want to insert a task somewhere other than at the bottom of your todo list, you need to use another syntax to create a task, a syntax which is based on `ed` commands. This is described below.

This is how you reorder tasks: 

$ t 3m0
0a1
> @harvardsq buy more moleskine notebooks
3d3
< @harvardsq buy more moleskine notebooks

$ t
     1	@harvardsq buy more moleskine notebooks
     2	@harvardsq buy some assam from tealuxe
     3	+opensource release todo.rb

The 3m0 is an ed script. That's right, todo.rb delegates a lot of commands to ed, the venerable Unix text editor. This gives you a lot of power to edit your todo list from the command line with very succinct commands, and without disrupting your command line workflow.

So if you wanted to delete the first task, you would use t 1d. If you wanted to delete a range, you could use something like t 1,3d.

If you want to insert text via an ed command sequence, put a space after the ed motion command, followed by the text to insert. Put quotes around the text if it contains any characters you don't want interpreted by the shell.

So for example, this would append a new task after line 3:

$ t 3a 'buy some fresh basil @traderjoes'

Remember, you do not have to use ed commands all the time. You can always open the todo.txt file and edit it directly in a more conventional text editor like Vim, Emacs, or TextEdit. You can also use Unix commands like cat >> todo.txt or echo 'buy arugla @wholefoods' >> todo.txt to add tasks. You have many options, and the data format of the todo.txt file is so simple that there is almost no way you can accidentally corrupt it. But it's possible to overwrite or delete the file accidentally, so please keep it backed up or under version control.

The "do" command marks tasks as done. 

$ t do 2
2d1
< @harvardsq buy some assam from tealuxe

$ t do /moleskine/
1d0
< @harvardsq buy more moleskine notebooks

You'll notice that in the second instance, we used a regular expression instead of a line number. This is possible because we are farming out work to `ed`. Tasks can be identified with ed-style addressing. So in the earlier example, instead of using t 3m0, we could have moved the task with t /notebook/m0.

What's the difference between t 2d and t do 2? The first command simply deletes a task. The second command moves it to the done.txt file and timestamps it with the date of completion. You can display done tasks with the "done" command. 

$ t done
     1	2012-01-29 @harvardsq buy some assam from tealuxe
     2	2012-01-29 @harvardsq buy more moleskine notebooks

To undo a task, use "undo" along with the line number of the done item. 

$ t undo 2
1a2
> @harvardsq buy more moleskine notebooks

$ t
     1	+opensource release todo.rb
     2  buy fresh basil @traderjoes
     3	@harvardsq buy more moleskine notebooks

You can filter tasks by tag like this: 

$ t @harvardsq
     3  @harvardsq buy more moleskine notebooks

You can also abbreviate tags. todo.rb will expand the abbreviated tag to the first tag it finds that matches the abbreviation. 

$ t @h
     3  @harvardsq buy more moleskine notebooks
You can list both incomplete and done tasks with the "all" command. You can pass a context or project tag to "all" to filter the list by that tag. Again, you can use abbrevation. 

$ t all @h
todo
     3	@harvardsq buy more moleskine notebooks

done
     1	2012-01-29 @harvardsq buy some assam from tealuxe

There is one more way to highlight and filter items, and that is by flagging tasks as high priority. A task is deemed to be high priority if it contains an exclamation mark. You can prioritize an action with the "pri" command and deprioritize it with the "depri" command. The "pri" command adds an exclamation mark to the end of the task and moves it to the top of the list. 

$ t pri /notebook/
0a1
> @harvardsq buy more moleskine notebooks!
3d3
< @harvardsq buy more moleskine notebooks

$ t
     1  @harvardsq buy more moleskine notebooks!
     2  +opensource release todo.rb
     3  buy fresh basil @traderjoes

If a task contains !!!, the item will actually flash. I'll spare you that demonstration.

To filter the list to show just priority items, use t ! or t ! followed by a tag or tag abbreviation:

$ t ! @h
     1  @harvardsq buy more moleskine notebooks!

To see a simple report of your tasks:

$ t report
tag            priority     todo     done
-------------- -------- -------- --------
+opensource           0        1        0
@harvardsq            1        1        1
@traderjoes           0        1        0

Extras

If you want to undo a destructive action, you can use the "revert" command:

t revert

Before changing todo.txt, todo.rb backs up todo.txt as .todo.txt.bkp. The revert command just swaps the two files. If you're versioning your todo.txt and done.txt files with Git, you can also use "git revert", "git stash" and similar commands to achieve the same effect.

If you don't want colored output, use the -C flag after t.

todo.rb also lets you customize colors. It will look for a colors.yml file in the current directory or in ~/.todo.rb/. The colors.yml file can specify the color for high priority items, default colors for context and project tags, as well as special colors for particular tags.

For example, this colors.yml

priority: fuchsia
context: FF69B4
project: FFEBCD
@harvardsq: DDAA00

would color the output like this: 

     1  +opensource release todo.rb
     2  buy fresh basil @traderjoes
     3  @harvardsq buy more moleskine notebooks!

Tips

I keep a global todo list in ~/todo/todo.txt and per-project todo.txt files in project folders. I use t as an alias for todo.rb to manage project todo.txt files. For managing the global todo.txt, I added this bash function to ~/.bash_profile to let me use `tt` to edit the global todo list from any directory: 

function tt()  {
  pushd ~/todo && t $@
  popd
}

You can use this same technique to wrap todo.rb in a bash function that does pre- and post-processing, such as pulling changes from an upstream git repository, committing new changes to git, and pushing those changes upstream.

function ttgit()  {
  git pull
  pushd ~/todo && t $@
  git commit -a -m "edit"  2> /dev/null
  git push
  popd
}

I synchronize the folder containing my todo.txt, done.txt, and colors.yml across my computers using a private upstream GitHub repository. You can also use Dropbox.

You can easily pipe todo.rb output into other Unix tools like head, grep, lpr, or mail.

For example, if you have a long list of tasks and you want to stay focused on the top items, you can do this: 

t | head -5

Because todo.rb is a standard, non-interactive Unix command-line program and todo.txt is a regular text file, you can go wild automating your information flows with Unix scripts.

All of the following scripts would be simple to write:

You can write crontasks to scan your todo.txt for items with dates and times in them and send you email reminders when things are due.

You can write a program to poll your Gmail inbox for emails matching certain rules and create tasks from the subject lines of those emails.

You can write a script to publish your todo list to a webserver and to automatically update it when the todo list changes. This lets you access your todo lists from any computer or mobile device, as well as share them with others.
